dua.com — Road to AI First

We built the machine that builds the product.

6 AI agents. One continuous loop. Every feature from idea to production — automatically.

Bogey Snoop Pixie Cyrus Docs Agent Patch

One Feature, Start to Finish

“Add a dark mode toggle to the settings screen.”

Step 01 — Intake

Bogey

Receives the idea, expands it into a product definition, creates 3 Linear tasks, and routes them to the appropriate lanes.

Design & Dev lanes run in parallel

Design Lane

1 Snoop researches dark mode patterns AI
2 Creates a design proposal AI
3 Designer finalizes in Figma Human
4 Pixie builds interactive prototype AI
5 Stakeholder green light Approved
6 Figma MCP → SwiftUI + Kotlin + React components AI

Dev Lane

1 GitNexus finds settings screen code AI
2 Reads MD files for context AI
3 Claude Code writes the toggle logic AI
4 Codex challenges the implementation AI
5 Mac Mini runs tests, opens PR AI
6 Code review & merge Human
← Both lanes merge →

Step 06 — Finalize

Documentation Agent

Runs CI, generates docs, creates the release, and sets a performance baseline.

Step 07 — Loop

Bogey picks up the next ticket

The cycle continues. Autonomously. Continuously.

01

Bogey

Task Creator

Takes raw ideas and half-formed feature requests, builds a product definition with scope, acceptance criteria, and routing logic. Creates Linear tasks and dispatches them to the right lane — design, dev, or both.

Tools

  • – Linear
  • – Product Definition
  • – Design Lane Routing
  • – Dev Lane Routing
Idea → Definition → Linear Tasks → Route
02

Snoop & Pixie

Design Agents

Snoop researches patterns, analyzes competitors, and generates a design proposal. Pixie takes the approved Figma design and builds interactive prototypes. Once stakeholders approve, the pipeline auto-generates SwiftUI, Kotlin, and React components from Figma via MCP.

Tools

  • – Idea + Prototype
  • – Green Light
  • – SwiftUI Components
  • – Kotlin Components
  • – React Components
Research → Proposal → Figma → Prototype → Components

Inspired by Uber’s design spec automation

03

Cyrus

Developer

The engineering backbone. GitNexus locates the relevant code, MD files provide architectural context, Claude Code writes the implementation, Codex reviews and challenges it. The Mac Mini runs the test suite and opens the PR. A human does the final review and merge.

Tools

  • – GitNexus
  • – MD Files
  • – Claude Code
  • – Codex
  • – Mac Mini
GitNexus → Context → Code → Review → Test & PR

Inspired by Stripe’s one-shot coding agents

04

Documentation Agent

CI, Docs & Release

Handles the final mile. Triggers CI pipelines, generates and updates documentation, automates release creation with changelogs, and establishes performance baselines for future regression checks.

Tools

  • – GitHub Actions
  • – Auto Documentation
  • – Release Automation
  • – Performance Baseline
CI → Docs → Release → Perf Baseline
05

Data Agent

Standalone — Analytics & BI

Operates independently from the main pipeline. Runs SQL queries, builds and maintains Tableau dashboards, performs exploratory data analysis, and delivers business intelligence insights on demand.

Tools

  • – SQL Query Engine
  • – Tableau Dashboards
  • – Data Exploration
  • – Business Intelligence
Query → Explore → Visualize → Insight
06

Patch / Support

Standalone — User Support

Handles the full support stack. In-app chat powered by Claude AI, automated FAQ resolution, premium human escalation when needed, and email fallback. Bugs get routed back to Bogey as Linear tickets. Changelog and feature requests close the feedback loop.

Tools

  • – In-App Chat
  • – Claude AI
  • – FAQ Automation
  • – Premium Human Escalation
  • – Email Fallback
  • – Linear Bugs → Bogey
  • – Changelog
  • – Feature Requests
Chat → AI Resolve → Escalate → Bogey

Inspired by Klarna’s AI support automation

And then it starts again.

The loop never stops
