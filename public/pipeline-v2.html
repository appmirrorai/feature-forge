:: AGENT REGISTRY
01 BOGEY // task creator

Takes raw ideas -- feature requests, bug reports, user feedback -- and transforms them into structured product definitions. Creates Linear tasks with full specs, acceptance criteria, and priority. Routes work to the design lane or dev lane depending on scope.

> role :: orchestrator -- the central brain that keeps the pipeline fed
> Linear
> Product Definition
> Design Lane Routing
> Dev Lane Routing
idea intake >> product definition >> linear tasks >> route to lane
02 SNOOP & PIXIE // design agents

Snoop handles research -- competitor analysis, design patterns, UX best practices. Creates a design proposal with detailed specs. After the human designer finalizes in Figma, Pixie converts approved frames into an interactive prototype. Once stakeholders approve, Figma MCP generates native SwiftUI, Kotlin, and React components automatically.

> inspired by :: how Uber automates design specs at scale
> Idea + Prototype
> Green Light Approval
> SwiftUI Components
> Kotlin Components
> React Components
research >> proposal >> figma [human] >> prototype >> approval [human] >> native components
03 CYRUS // developer

Runs on a dedicated Mac Mini. GitNexus locates the relevant codebase files while MD context files provide architecture rules and conventions. Claude Code writes the implementation. Codex reviews and challenges the code, surfacing edge cases. The Mac Mini runs the full test suite and opens a PR for human review.

> inspired by :: how Stripe builds one-shot coding agents
> GitNexus
> MD Files
> Claude Code
> Codex
> Mac Mini
gitnexus scan >> read context >> write code >> codex challenge >> test + PR >> review [human]
04 DOCS // documentation agent

The closer. After both lanes merge, the Documentation Agent verifies CI tests pass, generates updated docs from the code changes, handles release automation, and captures performance baselines. Once complete, it closes the ticket -- and Bogey picks up the next one.

> GitHub Actions
> Auto Documentation
> Release Automation
> Performance Baseline
CI tests >> auto docs >> release >> perf baseline >> close ticket
05 DATA // intelligence [standalone]

Operates independently from the main pipeline loop. Handles SQL queries against production data, powers Tableau dashboards, enables ad-hoc data exploration, and feeds business intelligence insights back into product decisions. Currently on standby.

> SQL Query Engine
> Tableau Dashboards
> Data Exploration
> Business Intelligence
sql queries >> dashboards >> exploration >> BI insights
06 PATCH // support agent [standalone]

The user-facing support layer. In-app AI chat powered by Claude handles the majority of requests. FAQ and knowledge base cover common questions. Premium users can escalate to a human. Email serves as a fallback channel. Bug reports flow back into Linear and route to Bogey. Public changelog and feature request intake close the feedback loop.

> inspired by :: how Klarna automated 2/3 of support with AI
> In-App AI Chat
> Claude AI
> FAQ / KB
> Premium Escalation
> Email Fallback
> Linear Bugs
> Public Changelog
> Feature Requests
user query >> ai chat >> faq/kb | human [premium] | email >> bugs to bogey
