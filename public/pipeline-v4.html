The Agents

Six specialized AI agents, each with their own tools and responsibilities.

01

Bogey — Task Creator

Takes raw ideas and product definitions, creates structured Linear tasks, and routes work to the correct lane — design or dev.

📋
Linear
Task creation, prioritization, and routing to the right lane.
📐
Product Definition
Scope, requirements, and acceptance criteria generation.
🎨
Design Lane
Routes to Snoop & Pixie for research and prototyping.
💻
Dev Lane
Routes to Developer Agent for implementation.
Idea → Product Definition → Linear Tasks → Route
02

Snoop & Pixie — Design Agents

End-to-end design pipeline: research, proposal, Figma mockups, interactive prototypes, stakeholder approval, and native code generation.

Inspired by Uber's design spec automation

💡
Idea + Prototype
Concept to validated design with research backing.
✅
Green Light
Stakeholder approval gate before development begins.
🍎
SwiftUI Components
Production-ready iOS components from Figma designs.
🤖
Kotlin Components
Android component generation from approved designs.
⚛️
React Components
Web component generation with design tokens.
Research → Proposal → Figma → Prototype → Approve → Code Gen
03

Cyrus — Developer

The engineering backbone. Uses GitNexus for AI-powered code discovery, reads MD files for context, writes implementation with Claude Code, and validates with Codex — all on a dedicated Mac Mini.

Inspired by Stripe's one-shot coding agents

🔍
GitNexus
AI-powered code findability across all repositories.
📄
MD Files
Project context, architecture decisions, and conventions.
⌨️
Claude Code
Writes the implementation — features, fixes, refactors.
🧪
Codex
Challenges and validates Claude's output for correctness.
🖥️
Mac Mini
Properly PC · Localghost · Always on, always building.
GitNexus → MD Context → Claude Code → Codex Review → Tests + PR
04

Documentation Agent — Ship & Record

Runs CI, auto-generates documentation, handles release automation, and records performance baselines. Then Bogey picks up the next ticket.

⚙️
GitHub Actions
CI/CD pipeline execution and monitoring.
📚
Auto Documentation
Generates and updates docs from code changes.
🚀
Release Automation
Version bumps, changelogs, and deployment triggers.
📈
Performance Baseline
Records metrics to detect regressions early.
CI Tests → Auto Docs → Release → Perf Baseline → Next Ticket
05

Data Agent — Standalone

Not part of the main pipeline. Handles SQL queries, Tableau dashboards, data exploration, and business intelligence on demand.

🗃️
SQL Query Engine
Natural language to SQL, query execution and results.
📊
Tableau Dashboards
Automated dashboard creation and data visualization.
🔬
Data Exploration
Anomaly detection, trend analysis, and data profiling.
💼
Business Intelligence
KPI tracking, cohort analysis, and executive reporting.
Question → SQL → Analysis → Dashboard
06

Support / Patch — Standalone

In-app AI chat for users. Handles FAQs, knowledge base lookups, premium human escalation, email fallback. Files bugs to Linear for Bogey. Manages changelogs and feature requests.

Inspired by Klarna's AI support automation

💬
In-App AI Chat
Claude-powered conversational support for users.
📖
FAQ / Knowledge Base
Auto-maintained KB with semantic search.
👤
Human Escalation
Premium tier escalation and email fallback.
🐛
Linear Bugs → Bogey
Files bugs and feature requests back into the pipeline.
User Message → AI Chat → Resolve / Escalate → Bug → Bogey

And then it starts again.
